# 扫码枪删除编码功能实现计划

## 1. 目标与现状分析
**目标**：在“编码管理”页面的操作栏（即“扫码添加”按钮旁边）添加一个“扫码枪删除”功能。用户点击按钮后弹出输入框，使用扫码枪扫描编码后自动查找并删除。如果未找到编码则提示用户。
**现状**：系统目前已有“快速添加”功能，支持扫码枪输入（通过 `onChange` 获取输入并在 `onPressEnter` 时提交）。我们需要实现一个类似逻辑的删除弹窗。

## 2. 具体修改方案

### 2.1 在编码管理页面增加“扫码删除”弹窗与逻辑
- **文件**：`client/src/pages/ProductDetail.js`
- **修改内容**：
  - **新增状态与引用**：
    - `scanDeleteModalVisible`: 控制“扫码删除”弹窗的显示。
    - `scanDeleteCode`: 记录弹窗输入框当前的值。
    - `scanDeleteInputRef`: 用于弹窗打开时和每次删除操作后自动聚焦输入框，以便连续扫码。
    - `scanDeleteLoading`: 控制删除操作时的加载状态（防止扫码枪连续回车导致重复提交）。
  - **新增处理函数 `handleScanDeleteSubmit`**：
    - 在输入框触发 `onPressEnter` 时调用。
    - 使用现有的 `extractCode` 提取有效编码，若为空则提示。
    - 查找 `codes` 中是否存在该编码。
    - 若不存在，调用 `message.error` 提示“未找到编码”，并清空输入框，保持聚焦。
    - 若存在，设置 `scanDeleteLoading(true)`，调用 `codeAPI.deleteCode(id, targetCode.id)`。
    - 删除成功后，调用 `loadCodes()`，提示“删除成功”，清空输入框，保持聚焦，最后取消加载状态。
  - **更新 UI (按钮)**：
    - 在 `Card` 组件的 `extra` 操作区（“扫码添加”按钮左侧），新增一个带有 `danger` 属性的“扫码删除”按钮。
  - **新增 UI (弹窗)**：
    - 新增一个 `Modal` 组件，绑定 `scanDeleteModalVisible`，隐藏默认 footer。
    - 弹窗内包含一个提示信息（说明操作流程）和一个 `Input` 输入框。
    - 输入框绑定 `value={scanDeleteCode}`，`onChange={e => setScanDeleteCode(e.target.value)}`，`onPressEnter={handleScanDeleteSubmit}`，并支持 `autoFocus`（或在弹窗打开时通过 `useEffect` 自动聚焦）。

## 3. 假设与决策
- **决策 1**：依据系统记忆 `Barcode Scanner Input Handling Strategy`，扫码枪输入过程（`onChange`）不做任何字符截断或处理，所有清理与匹配逻辑（`extractCode`）均在按下回车（`onPressEnter`）后执行，确保兼容各种扫码枪的输入速度。
- **决策 2**：为支持连续扫码删除，提交后无论成功还是失败，均自动清空输入框并重新聚焦。
- **假设**：弹窗中的删除操作是对当前产品下**未被删除（未进入回收站）**的编码。如果被扫码枪录入的编码已在回收站中或不属于当前产品，系统会提示“未找到编码”。

## 4. 验证步骤
1. 打开“产品详情 -> 编码管理”页面。
2. 确认页面右上角出现了红色的“扫码删除”按钮。
3. 点击“扫码删除”，弹出带有输入框的“扫码枪删除编码”弹窗，且输入框自动获得焦点。
4. 模拟扫码枪输入一个已存在的编码并回车，确认系统提示删除成功，输入框被清空且继续保持焦点，同时编码从后台列表中消失。
5. 模拟扫码枪输入一个不存在的编码并回车，确认系统提示“未找到编码”，输入框被清空并继续保持焦点。
